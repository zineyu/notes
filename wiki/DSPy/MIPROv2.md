# MIPROv2：联合优化指令和例题

## 概述
同时解决"怎么写更好指令"和"怎么搭配指令和例题"两个问题。先用 LLM 自动生成高质量指令和例题候选，再用贝叶斯优化（Optuna TPE）搜索最佳组合。类似「淘宝选商品」的推荐思路。

## 三阶段流程

### 阶段1：生成候选例题
本质是 **BootstrapFewShot 的批量执行**，但目的不是直接选出最优，而是生成足够多的「例题套餐」供后续搜索。

生成策略：
- 方案 A：零样本（不给例题）
- 方案 B：标注例题（直接从标注数据取）
- 方案 C：顺序 Bootstrap（按原始顺序收集）
- 方案 D-N：随机 Shuffle + 随机例题数量

每组候选 = 程序中**每个 predictor** 各配一套例题。

### 阶段2：生成候选指令

通过 **7 个 Anchor（锚点）** 将 LLM 输出锚定在任务真实特征上：

| Anchor | 是什么 | 为什么有用 |
|--------|--------|-----------|
| **数据集摘要** | 训练集的 2-3 句话概括 | LLM 知道数据主题、格式、常见模式 |
| **程序源码** | DSPy 程序的 Python 代码 | 理解程序结构，知道每个 predictor 职责 |
| **模块角色描述** | LLM 自动分析"程序是做什么的" | 理解整体任务上下文 |
| **高质量例题** | 阶段1 收集的成功例题 | 提供具体输入输出样例 |
| **历史尝试记录** | 之前试过哪些指令、得分如何 | 迭代改进，避免重复失败 |
| **随机风格提示** | "be creative" / "include persona" 等 | 探索不同风格指令空间 |
| **你的原始指令** | Signature 里的 docstring | 作为修改起点，确保不偏离意图 |

> 每次生成指令时随机选一个风格提示（Tip），相当于给 LLM 换不同「角色设定」。

### 阶段3：贝叶斯优化搜索

**搜索空间示例**：3 个 predictor，各 10 条指令 × 8 组例题 = 80 种组合 → 总空间 80³ = **512,000**。

直接穷举不现实，用贝叶斯优化的 TPE 算法高效搜索，考虑参数间联合分布。

**评估策略：Minibatch + Full eval 交替**

| 评估层级 | 样本数 | 目的 | 精度 |
|----------|--------|------|------|
| **小批量** | 35 | 快速筛选，淘汰明显差的 | 有噪声但便宜 |
| **全量** | 完整验证集 | 精确评估最有潜力的组合 | 准确但昂贵 |

每 5 轮小批量评估后，对得分最高的组合做一次全量评估，结果反馈给贝叶斯优化器。

## 设计亮点

### Prompt Model 与 Task Model 分离
- **Prompt Model**（如 GPT-4）：需要创意和深度理解，生成指令和数据摘要
- **Task Model**（如 GPT-4o-mini）：需要大量重复评估，用便宜模型
- 生产环境重要的成本优化策略

### 迭代式数据集摘要
分批看数据、逐步补充观察，避免单次观察偏差。

### 默认基线注入
优化开始前将当前程序的零样本配置作为 trial 注入，确保结果**不会比什么都不做更差**。

## 关键参数

| 参数 | 作用 | 建议 |
|------|------|------|
| `auto` | 预设档位 light/medium/heavy | light 快速验证，medium 平衡，heavy 追求极限 |
| `prompt_model` | 生成指令的模型 | 可用更强模型 |
| `task_model` | 评估候选的模型 | 可用更便宜模型节省成本 |
| `num_candidates` | 每个 predictor 的候选数量 | light≈6, medium≈12, heavy≈18 |
| `minibatch_size` | 小批量评估样本数 | 默认 35，数据量大可增大 |

## 适用场景

```
数据量?
  <50 条   → BootstrapFewShot（简单快速）
  50-200 条 → BootstrapFewShotWithRandomSearch
  200+ 条  → 需要优化指令? → 否 → BootstrapFewShotWithOptuna
                          → 是 → MIPROv2
                              预算充足? → 是 → heavy 模式（~100+ trials）
                                        → 否 → medium/light 模式
```

## 与 BootstrapFewShot 的对比

| 维度 | BootstrapFewShot | MIPROv2 |
|------|-----------------|---------|
| 优化对象 | 仅例题 | **指令 + 例题** 联合优化 |
| 搜索策略 | 无搜索，直接收集 | **贝叶斯优化** |
| 指令来源 | 原始 docstring | **LLM 自动生成**多种候选 |
| 数据利用 | 筛选正确 trace | 数据集摘要 + 程序源码分析 |
| 评估方式 | 编译即结束 | 小批量 + 全量 交替 |
| 数据量 | <50 条 | **200+ 条**发挥优势 |
| 计算成本 | 低 | 高（数十到数百次 LLM 调用） |
